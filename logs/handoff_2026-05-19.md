## Layer 4.1 priority bug — WARNING explícito

O scanner v2.8/2.9 com `foil=False` prioriza preços TCG na ordem `reverseHolofoil > normal` quando ambos existem. Para sets vintage (e-Card era, Wizards era) **o reverseHolofoil costuma ter prêmio enorme sobre o normal** (cards holo-rare reprintadas em reverse holo são raras). Isso pode **inflar artificialmente o TCG R$** dos top deals atuais.

**Exemplos suspeitos no Top 20 (ticket #4 do operador):**
- Possíveis cards vintage onde o preço TCG mostrado pode estar pegando o reverseHolofoil em vez do normal: **Hitmonlee 013/110 LC, Nidoqueen 032/110 LC, Pidgeot 033/110 LC, Pokémon Trader 103/110 LC, Scoop Up 104/110 LC, Feraligatr 046 Expedition, Blissey/Togetic/Vileplume Aquapolis**.
- Tick #4 referência: Snorlax LC R$ 3.275 / Alakazam Skyridge R$ 1.219 / Mantine Skyridge R$ 604 (todos vintage com reverseHolofoil inflado).

**Patch sugerido para v2.10:**
Trocar ordem de priority em `cardtrader_scanner.py` (Layer 4.1) de:
```
order = ['reverseHolofoil', 'holofoil', 'normal']  # atual (errado para foil=False)
```
para:
```
order = ['normal', 'reverseHolofoil', 'holofoil']  # foil=False prefere normal genuíno
```
Quando o listing CT é foil normal (não-reverse), o TCG normal price é a comparação correta. Aplicar para todos os caminhos onde `foil_flag is False`.
